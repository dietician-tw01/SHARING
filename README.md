# 2026 周年慶限定優惠 — 滿額贈品試算頁

一個單頁 HTML 工具，輸入各項商品購買數量，即時試算，請注意，此工具限定內部使用，不可公開給消費者，案型為2026周年慶案型
- 依官網價格自動計算單價、總計
- 累計總金額
- 依總金額對應目前可獲得的滿額贈品
- 顯示「還差多少錢」可以升級到下一階贈品門檻

純前端、無後端、無需安裝，開啟 `index.html`（或本檔）即可使用。

---

## 檔案結構

```
.
└── index.html   # 單一檔案，包含 HTML / CSS / JavaScript，無外部相依套件
```

商品圖片透過外部圖床連結載入（`https://blog.dietician.com.tw/...`、`https://dietician-public-static.s3...`），需要網路連線才能正常顯示圖片，其餘功能（試算、贈品判斷）不受影響。

## 使用方式

1. 直接用瀏覽器開啟 HTML 檔案（雙擊或拖曳到瀏覽器視窗）即可使用，不需要伺服器或建置流程。
2. 也可以放到任何靜態網站空間（GitHub Pages、S3、Netlify 等）直接對外提供給客戶使用。
3. 頁面操作：
   - 每個商品卡片可用「+ / -」按鈕或直接輸入數字調整購買數量
   - 上方橫幅即時顯示目前總額，以及距離下一個贈品門檻還差多少錢
   - 底部固定列顯示目前符合資格的贈品與應付總額
   - 「清空重置」按鈕可將所有商品數量歸零，並捲動回頁面頂端

## 定價邏輯（單價級距）

每項商品在 `pricingRules` 中設定 4 階單價（陣列依序對應以下數量門檻）：

| 購買數量 | 套用單價 |
|---|---|
| 1 件 | `prices[0]`（原價） |
| 2–3 件 | `prices[1]` |
| 4–9 件 | `prices[2]` |
| 10 件以上 | `prices[3]`（最低價） |

邏輯實作於 `getUnitPrice()`：數量越多，適用陣列中越後面（越低）的單價。小計 = 該數量對應單價 × 數量，加總所有商品小計即為總金額。

## 如何修改 / 客製化

### 新增或調整商品
編輯 `pricingRules` 物件，新增一筆：
```js
"商品名稱": { prices: [原價, 2-3件單價, 4-9件單價, 10件以上單價], img: "圖片網址" }
```
新增後頁面會自動產生對應的商品卡片，不需要另外改 HTML。

### 調整贈品門檻或內容
編輯 `giftRules` 陣列，新增／修改任一筆 `{ amount: 門檻金額, gift: "贈品說明文字" }` 即可，程式會自動依金額排序判斷，不需手動維持順序。

### 調整數量級距（目前為 1 / 2–3 / 4–9 / 10+）
修改 `getUnitPrice()` 函式中的判斷條件（`qty >= 10`、`qty >= 4`、`qty >= 2`）。

## 注意事項

- 此頁面**完全在瀏覽器端運算**，不會將任何資料送出到伺服器，也不會保存使用者輸入（重新整理頁面後數量會歸零）。
- 若要上線給外部客戶使用，建議：
  - 確認商品圖片外部連結長期有效
  - 活動結束後記得下架或更新頁面標題／規則，避免舊優惠繼續曝光
- 若日後此工具會長期使用或有專人維護，建議依公司內部規範登記到工具登記表（負責人、風險等級、最後驗證日期），並考慮改用組織 GitHub repo 管理版本，方便交接與追蹤異動。
